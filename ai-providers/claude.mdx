Claude by Anthropic offers powerful AI models known for their nuanced understanding and thoughtful responses, perfect for generating detailed meeting summaries.

## Overview

Claude is Anthropic's AI assistant, featuring:
- **Advanced reasoning** for complex meeting analysis
- **Large context windows** (up to 200K tokens)
- **Safety-focused** design with built-in guardrails
- **Nuanced understanding** of context and tone

**Best for:** Detailed analysis, complex meetings, executive summaries

## Prerequisites

<Card title="Anthropic Account Required" icon="user">
  You need an active Anthropic account with API access to use Claude with Meetily.
</Card>

## Setup

<Steps>
  <Step title="Create Anthropic Account">
    Visit [console.anthropic.com](https://console.anthropic.com) and sign up for an account.
  </Step>

  <Step title="Obtain API Key">
    1. Navigate to **API Keys** in the Anthropic Console
    2. Click **Create Key**
    3. Name your key (e.g., "Meetily")
    4. Copy the API key (starts with `sk-ant-`)
    
    <Warning>
      Store your API key securely. It won't be shown again after creation.
    </Warning>
  </Step>

  <Step title="Configure in Meetily">
    1. Open **Settings > Summary** in Meetily
    2. Select **Claude** from the Summarization Model dropdown
    3. An API Key field will appear
  </Step>

  <Step title="Enter API Key">
    1. Paste your API key into the **API Key** field
    2. Click the lock icon to secure it
    3. Select your preferred Claude model
    4. Click **Done** to save
  </Step>
</Steps>

## API Key Management

### Storing Your API Key

Meetily stores your API key securely in your local database (SQLite) with these security features:

1. **Local storage only** - Never transmitted except to Anthropic's API
2. **Database encryption** - Stored in encrypted database fields
3. **Lock protection** - UI lock prevents accidental modification

### Updating Your API Key

<Steps>
  <Step title="Unlock the Field">
    In Settings > Summary, click the **lock icon** next to the API Key field.
  </Step>

  <Step title="Update Key">
    Paste your new API key and click the **lock icon** to secure it.
  </Step>

  <Step title="Save Changes">
    Click **Done** to save the updated configuration.
  </Step>
</Steps>

### Viewing Your API Key

1. Navigate to **Settings > Summary**
2. Select **Claude** as the provider
3. Click the **eye icon** to toggle visibility
4. The key will be shown temporarily (masked by default)

## Usage Examples

### Generate a Meeting Summary

1. **Record** your meeting in Meetily
2. **Stop** the recording when complete
3. **Auto-summary** will trigger if enabled
4. Claude will:
   - Analyze the full transcript
   - Extract key points and decisions
   - Identify action items
   - Generate a structured summary

### Manual Summary Generation

1. Navigate to an existing meeting
2. Click **Generate Summary**
3. Select Claude as the provider (if not default)
4. Wait for processing (~10-30 seconds depending on meeting length)
5. View the generated summary

## Troubleshooting

### Invalid API Key

**Error:** "Invalid API key" or "Authentication failed"

**Solution:**
1. Verify the API key starts with `sk-ant-`
2. Check for extra spaces or line breaks
3. Ensure the key hasn't been revoked in Anthropic Console
4. Generate a new API key if needed

### Rate Limit Errors

**Error:** "Rate limit exceeded"

**Solution:**
1. **Free tier limits** - Upgrade to a paid plan for higher limits
2. **Wait and retry** - Rate limits reset after a period
3. **Reduce frequency** - Avoid generating multiple summaries simultaneously

### Model Not Available

**Error:** "Model not found" or model missing from dropdown

**Solution:**
1. **Check API key permissions** - Some models require specific access
2. **Refresh models** - Switch providers and back to Claude to refresh
3. **Update Meetily** - Newer versions support latest models
4. **Use fallback models** - If API fetch fails, fallback models are shown

### Slow Response Times

**Issue:** Summary generation takes longer than expected

**Cause:**
- Very long meetings (>2 hours)
- High API load during peak times
- Using Opus model (slower but highest quality)

**Solution:**
1. **Switch to Haiku or Sonnet** for faster results
2. **Split long meetings** - Record in segments if possible
3. **Off-peak processing** - Generate summaries during low-traffic hours

### Connection Errors

**Error:** "Failed to connect to Anthropic API"

**Solution:**
1. **Check internet connection**
2. **Verify firewall settings** - Allow HTTPS access to `api.anthropic.com`
3. **Check Anthropic status** - Visit [status.anthropic.com](https://status.anthropic.com)
4. **Retry** - Temporary network issues may resolve
