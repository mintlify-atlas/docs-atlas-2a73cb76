OpenAI provides industry-leading GPT models for generating high-quality meeting summaries with excellent reasoning and natural language understanding.

## Overview

OpenAI's GPT models offer:
- **Cutting-edge performance** with GPT-4 and GPT-5 series
- **Large context windows** (up to 128K tokens)
- **Reasoning models** (o1, o3 series) for complex analysis
- **Fast inference** with optimized infrastructure
- **Multimodal capabilities** (future: analyze meeting screenshots)

**Best for:** High-quality summaries, complex reasoning, fast results

## Setup

<Steps>
  <Step title="Create OpenAI Account">
    1. Visit [platform.openai.com](https://platform.openai.com)
    2. Sign up or log in to your account
    3. Add payment method (required for API access)
  </Step>

  <Step title="Generate API Key">
    1. Navigate to **API Keys** in your OpenAI Dashboard
    2. Click **Create new secret key**
    3. Name your key (e.g., "Meetily")
    4. Set permissions (default is fine)
    5. Copy the API key (starts with `sk-`)
    
    <Warning>
      Save your API key immediately. It won't be displayed again.
    </Warning>
  </Step>

  <Step title="Configure in Meetily">
    1. Open **Settings > Summary** in Meetily
    2. Select **OpenAI** from the Summarization Model dropdown
    3. The API Key field will appear
  </Step>

  <Step title="Enter API Key">
    1. Paste your API key into the **API Key** field
    2. Click the lock icon to secure it
    3. Select your preferred GPT model
    4. Click **Done** to save
  </Step>
</Steps>

## API Key Management

### Storing Your API Key

Meetily stores your OpenAI API key securely:

1. **Local database storage** - Saved in SQLite database
2. **Encrypted fields** - API keys are stored in encrypted columns
3. **No cloud sync** - Keys never leave your machine except for API calls
4. **Lock protection** - UI prevents accidental editing

### Security Features

```typescript
// API key is only used for:
// 1. Fetching available models (optional)
// 2. Generating summaries via OpenAI API

// API key is NEVER:
// - Logged or printed
// - Sent to Meetily servers
// - Shared with third parties
```

- **1-hour meetings** typically fit entirely
- **2-3 hour meetings** may require chunking

- 4-hour meeting: ~40,000 tokens (may need chunking)

## Troubleshooting

### Invalid API Key

**Error:** "Invalid API key" or "Incorrect API key provided"

**Solution:**
1. Verify the key starts with `sk-`
2. Check for extra spaces or line breaks when pasting
3. Ensure the key hasn't been revoked in OpenAI Dashboard
4. Generate a new key if needed

### Insufficient Credits

**Error:** "You exceeded your current quota"

**Solution:**
1. **Add payment method** in OpenAI Dashboard
2. **Check billing** - Ensure you have available credits
3. **Upgrade plan** if on free trial (free trial is limited)
4. **Wait for reset** if you hit rate limits

### Rate Limit Errors

**Error:** "Rate limit reached" or "429 Too Many Requests"

**Solution:**
1. **Tier-based limits** - Higher usage tiers get higher rate limits
2. **Wait and retry** - Rate limits reset after a period (usually per minute)
3. **Upgrade tier** - Increase your usage tier in OpenAI Dashboard
4. **Batch processing** - Avoid generating multiple summaries simultaneously

### Model Not Available

**Error:** Model missing from dropdown or "model not found"

**Solution:**
1. **Check access** - Some models require special access (e.g., o-series)
2. **Refresh models** - Switch providers and back to OpenAI to refresh the list
3. **Use fallback** - If API fetch fails, fallback models are shown
4. **Update Meetily** - Ensure you're on the latest version

### Connection Issues

**Error:** "Failed to connect to OpenAI API"

**Solution:**
1. **Check internet connection**
2. **Verify firewall** - Allow HTTPS to `api.openai.com`
3. **Check OpenAI status** - Visit [status.openai.com](https://status.openai.com)
4. **Use VPN** - If in a region with restrictions, use a VPN

### Slow Performance

**Issue:** Summary takes longer than expected

**Possible causes:**
- Using reasoning models (o1/o3) which are slower
- Very long meetings requiring chunking
- High API load during peak hours

**Solution:**
1. **Switch to faster models** - Use `gpt-4o-mini` instead of reasoning models
2. **Off-peak processing** - Generate summaries during low-traffic times
3. **Check status page** - Verify no ongoing incidents

## Advanced Features

### Dynamic Model Fetching

Meetily automatically fetches available models based on your API key:

```typescript
// Backend code fetches and caches models
const response = await fetch('https://api.openai.com/v1/models', {
  headers: {
    'Authorization': `Bearer ${apiKey}`
  }
});

// Filters to show only chat models:
// - gpt-* series
// - o1-*, o3-*, o4-* reasoning models
// Excludes: embeddings, TTS, Whisper, DALL-E
```

### Model Filtering

The backend automatically filters models to show only those suitable for chat:

```rust
fn is_chat_model(model_id: &str) -> bool {
    let id = model_id.to_lowercase();
    (id.starts_with("gpt-") ||
     id.starts_with("o1-") ||
     id.starts_with("o3-") ||
     id.starts_with("o4-"))
    && !id.contains("embedding")
    && !id.contains("tts")
    && !id.contains("whisper")
    && !id.contains("dall-e")
}
```
