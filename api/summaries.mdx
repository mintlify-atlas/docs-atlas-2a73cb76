## Process Transcript

<CodeGroup>
```bash cURL
curl -X POST http://localhost:5167/process-transcript \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Full transcript text here...",
    "model": "claude",
    "model_name": "claude-3-5-sonnet-20241022",
    "meeting_id": "meeting-1704067200000",
    "chunk_size": 5000,
    "overlap": 1000,
    "custom_prompt": "Focus on action items and decisions"
  }'
```

```python Python
import requests

data = {
    "text": "Full transcript text here...",
    "model": "claude",
    "model_name": "claude-3-5-sonnet-20241022",
    "meeting_id": "meeting-1704067200000",
    "chunk_size": 5000,
    "overlap": 1000,
    "custom_prompt": "Focus on action items and decisions"
}

response = requests.post(
    "http://localhost:5167/process-transcript",
    json=data
)
result = response.json()
process_id = result["process_id"]
```

```typescript TypeScript
const response = await fetch('http://localhost:5167/process-transcript', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    text: 'Full transcript text here...',
    model: 'claude',
    model_name: 'claude-3-5-sonnet-20241022',
    meeting_id: 'meeting-1704067200000',
    chunk_size: 5000,
    overlap: 1000,
    custom_prompt: 'Focus on action items and decisions'
  })
});

const result = await response.json();
const processId = result.process_id;
```
</CodeGroup>

Start AI-powered summarization of a meeting transcript. This endpoint processes the transcript in chunks using the specified AI model and generates structured summaries including action items, key decisions, and meeting notes.

<Note>
  This is an **asynchronous operation**. The endpoint returns immediately with a `process_id`. Use the [Get Summary](#get-summary) endpoint to check the status and retrieve results.
</Note>

### Request Body

<ParamField body="text" type="string" required>
  The complete transcript text to process (max 10MB)
</ParamField>

<ParamField body="model" type="string" required>
  AI provider to use: `"claude"`, `"ollama"`, `"groq"`, or `"openai"`
</ParamField>

<ParamField body="model_name" type="string" required>
  Specific model name for the provider:
  - Claude: `claude-3-5-sonnet-20241022`, `claude-3-opus-20240229`
  - Ollama: `llama3`, `phi4`, `qwen2.5`, etc.
  - Groq: `llama-3.3-70b-versatile`, `mixtral-8x7b-32768`
  - OpenAI: `gpt-4o`, `gpt-4o-mini`, `gpt-4-turbo`
</ParamField>

<ParamField body="meeting_id" type="string" required>
  The meeting ID to associate the summary with
</ParamField>

<ParamField body="chunk_size" type="integer" default={5000}>
  Number of characters per chunk (adjusted automatically for Ollama models)
</ParamField>

<ParamField body="overlap" type="integer" default={1000}>
  Number of characters to overlap between chunks (ensures context continuity)
</ParamField>

<ParamField body="custom_prompt" type="string" default="Generate a summary of the meeting transcript.">
  Additional instructions for the AI model
</ParamField>

### Response

<ResponseField name="message" type="string" required>
  Status message (always `"Processing started"`)
</ResponseField>

<ResponseField name="process_id" type="string" required>
  The meeting ID to use for checking status (same as input `meeting_id`)
</ResponseField>

<ResponseExample>
```json 200 - Processing Started
{
  "message": "Processing started",
  "process_id": "meeting-1704067200000"
}
```

```json 500 - Error
{
  "detail": "ANTHROPIC_API_KEY environment variable not set"
}
```
</ResponseExample>

## Get Summary

<CodeGroup>
```bash cURL
curl http://localhost:5167/get-summary/meeting-1704067200000
```

```python Python
import requests
import time

meeting_id = "meeting-1704067200000"

# Poll for completion
while True:
    response = requests.get(f"http://localhost:5167/get-summary/{meeting_id}")
    result = response.json()
    
    if result["status"] == "completed":
        print("Summary ready!")
        print(result["data"])
        break
    elif result["status"] == "error":
        print(f"Error: {result['error']}")
        break
    else:
        print("Processing...")
        time.sleep(2)
```

```typescript TypeScript
const meetingId = 'meeting-1704067200000';

// Poll for completion
while (true) {
  const response = await fetch(`http://localhost:5167/get-summary/${meetingId}`);
  const result = await response.json();
  
  if (result.status === 'completed') {
    console.log('Summary ready!', result.data);
    break;
  } else if (result.status === 'error') {
    console.error('Error:', result.error);
    break;
  } else {
    console.log('Processing...');
    await new Promise(r => setTimeout(r, 2000));
  }
}
```
</CodeGroup>

Retrieve the status and results of a summary processing job.

### Path Parameters

<ParamField path="meeting_id" type="string" required>

<ResponseField name="status" type="string" required>
  Processing status: `"processing"`, `"completed"`, or `"error"`

<ResponseField name="data" type="object">
  The structured summary data (only present when status is `"completed"`)

    <ResponseField name="_section_order" type="array">

      <Expandable title="Section Object">
        <ResponseField name="title" type="string">
          Section display title
        </ResponseField>
        
        <ResponseField name="blocks" type="array">
          Array of content blocks
          
          <Expandable title="Block Object">

              Unique block identifier
            </ResponseField>
            
            <ResponseField name="type" type="string">
              Block type: `"text"`, `"bullet"`, `"heading1"`, `"heading2"`

            <ResponseField name="content" type="string">
              Block text content
            </ResponseField>
            
            <ResponseField name="color" type="string">
              Color hint: `"gray"` or empty string for default
            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="error" type="string">
      Error message (only present when status is `"error"`)
    </ResponseField>
  </Expandable>
</ResponseField>

## How AI Summarization Works

### Chunking Strategy

Long transcripts are split into overlapping chunks to fit within model context windows:

```
Transcript (20,000 chars)
│
├─ Chunk 1: chars 0-5000
├─ Chunk 2: chars 4000-9000 (1000 char overlap)
├─ Chunk 3: chars 8000-13000
└─ Chunk 4: chars 12000-17000
```

**Why overlapping?** Ensures no information is lost at chunk boundaries.

### Structured Output

The AI models are prompted to return structured JSON with predefined sections:

    Participants and their roles
  </Card>
  <Card title="Session Summary" icon="file-lines">
    High-level overview of the meeting
  </Card>
  <Card title="Key Items & Decisions" icon="lightbulb">
    Important decisions made
  </Card>
  <Card title="Immediate Action Items" icon="list-check">
    Tasks with owners and deadlines
  </Card>

    Follow-up actions and future plans
  </Card>
  <Card title="Critical Deadlines" icon="calendar-clock">
    Time-sensitive items
  </Card>
</CardGroup>

### Model-Specific Behavior

| Provider | Chunk Size | Notes |
|----------|------------|-------|
| **Claude** | 5,000 chars | Uses Pydantic AI structured output |
| **OpenAI** | 5,000 chars | Uses Pydantic AI structured output |
| **Groq** | 5,000 chars | Uses Pydantic AI structured output |
| **Ollama (Llama/Phi)** | 10,000 chars | Smaller context, adjusted automatically |
| **Ollama (Other)** | 30,000 chars | Larger context window models |

<Tip>
  For best results with Ollama, use models with larger context windows like `qwen2.5:32k` or `llama3.1:70b`.
</Tip>

## Error Handling

### Common Errors

<AccordionGroup>
  <Accordion title="API Key Not Configured">
    **Error:** `"Anthropic API key not configured. Please set your API key in the model settings."`
    
    **Solution:** Save your API key using the [Save Model Config](#save-model-config) endpoint or configure it in the frontend settings.
  </Accordion>
  
  <Accordion title="Empty Transcript">
    **Error:** `"Empty transcript text provided"`
    
    **Solution:** Ensure the `text` field contains actual transcript content.
  </Accordion>
  
  <Accordion title="Invalid Chunk Parameters">
    **Error:** `"chunk_size must be positive"`
    
    **Solution:** Use positive values for `chunk_size` and non-negative values for `overlap`.
  </Accordion>
  
  <Accordion title="Meeting Not Found">
    **Error:** `"Meeting ID not found"`
    
    **Solution:** Ensure the meeting was created using the [Save Transcript](/api/transcripts#save-transcript) endpoint before processing.
  </Accordion>
</AccordionGroup>

## Best Practices

<Steps>
  <Step title="Save Transcripts First">
    Always save transcripts using the `/save-transcript` endpoint before processing. This creates the meeting record.
  </Step>
  
  <Step title="Choose the Right Model">
    - **Quick testing**: Use Ollama with a small model like `phi4`
    - **Best quality**: Use Claude (`claude-3-5-sonnet-20241022`)
    - **Fast API**: Use Groq (`llama-3.3-70b-versatile`)
  </Step>
  
  <Step title="Poll Intelligently">
    Check status every 2-5 seconds. Processing typically takes 10-60 seconds depending on transcript length and model.
  </Step>
  
  <Step title="Use Custom Prompts">
    Add context with `custom_prompt` to get summaries tailored to your needs:
    ```json
    {
      "custom_prompt": "Focus on technical decisions and code review feedback"
    }
    ```
  </Step>
</Steps>
